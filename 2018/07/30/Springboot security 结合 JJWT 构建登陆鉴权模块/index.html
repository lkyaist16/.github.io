Springboot security 结合 JJWT 构建登陆鉴权模块

发表于 2018-07-30

介绍

  最近在学习Springboot和Springcloud构建微服务架构项目，一直对项目中网关安全控制和权限鉴定，这部分内容理解不是很深，决定自己也动手搭建一个拥有鉴权与安全控制的Springcloud网关。

开始

  首先，这是一个Springcloud+Springboot+Mybatis搭建的基础网关微服务项目，通过在配置文件中定义springcloud-zuul可以访问到在同一个注册中心中注册的其他服务，集成了负载均衡ribbon和容错保护hystrix功能，其次还继承了redis数据库对登录后的token进行持久化，集成swagger2对接口功能进行调试。这里主要记录的是springboot-security和JJWT的整合搭建过程和一些踩过的坑。

  实际我们开发的web项目中，http请求先要通过gateway网关，进行登录验证，并且调用其他服务接口时，需要对其权限进行鉴定，只有拥有权限的用户才可以调用。这时，我们就需要引入Springboot-security和JJWT来对请求进行过滤与鉴权。
  首先，Springboot-security其本质是Spring-security安全控制框架，其开箱即用的特性，让我们减少了很多复杂的xml配置。实际上，security中定义了很多过滤器，对http请求进行拦截过滤来实现请求的安全控制。在springboot-security里我们只需要配置一个WebSecurityConfig类就可以实现安全控制的功能。

这样除了可以匿名访问的地址以外，其他所有请求都要通过鉴权认证。这时我们引入JJWT(Java Json Web Token)关于jjwt这里就不多作介绍了，是一个非常轻量级的鉴权验证工具。

  每次登录验证通过后，会生成一个token，并持久化到redis里，之后每次请求都要先通过一个JwtAuthenticationTokenFiter类，将请求头中的token与redis里的token进行验证，并且设置了过期时间，判断是否过期。
